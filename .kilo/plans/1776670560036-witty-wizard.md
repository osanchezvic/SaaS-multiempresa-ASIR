# Plan: Security Analysis and Authelia Integration Assessment

## Objective
Perform a deep analysis of the "SaaS-multiempresa-ASIR" codebase to identify code and security vulnerabilities, and evaluate the feasibility/impact of integrating Authelia for enhanced security.

## Completed Analysis
- **Bash Scripts:** Identified insecure plaintext credential storage (`scripts/funciones/utils.sh`) and command injection risks via shell variable interpolation in templates.
- **Admin Dashboard:** Found potential CSRF vulnerability on logout, lack of CSRF protection in login/actions, and hardcoded default credentials.
- **Infrastructure:** Identified hardcoded passwords in `infra/proxy/docker-compose.yml` (`npm123`) and exposed management ports (81) without secondary authentication.

## Implementation Roadmap (Authelia Integration)
1. **Infrastructure Hardening:**
   - Remove hardcoded credentials from `infra/proxy/docker-compose.yml` and replace with environment variable references (`.env`).
   - Implement CSRF protection for `admin-dashboard/index.php`.
2. **Authelia Deployment:**
   - Deploy `authelia` and `redis` (session storage) in `infra/docker-compose.yml`.
   - Configure Nginx Proxy Manager to use "Forward Auth" against Authelia for critical services (`admin-dashboard` and NPM management console).
3. **Security Refactoring:**
   - Transition credential management in `scripts/` to use a more secure storage or encrypted files.

## Timeline/Steps
1.  **Hardening:** Update `infra/proxy/docker-compose.yml` to use `.env` files for secrets.
2.  **Authelia Integration:** Modify `infra/docker-compose.yml` to include Authelia containers.
3.  **Proxy Configuration:** Update NPM configurations to point to Authelia.
4.  **PHP Updates:** Add CSRF tokens to `infra/admin-dashboard/index.php`.
5.  **Final Review:** Verify security posture of new configuration.
